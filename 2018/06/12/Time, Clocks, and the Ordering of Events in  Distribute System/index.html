分布式原理学习（四）： Time, Clocks, and the Ordering of Events in Distribute System

2018-06-12

Partial Ordering

假设一个系统由多个进程组成，每个进程包含一系列事件。接收或发送消息也是进程中的一个事件。我们用⟶表示‘happend before’关系，定义如下：

Definition: 关系⟶需要满足以下三个条件：

（1）如果事件a和b是在同一进程中，并且a先于b发生，那么a⟶b。

（2）一个进程发送消息的事件为a，另一个接收同一消息的事件为b，那么a⟶b。

（3）如果a⟶b，b⟶c，那么a⟶c。

如果a ↛ b ∧ b ↛ a，那么a和b是并发的。

对于任意事件，a ↛ a。这是因为一个事件不能先于它自身发生。

Logical Clocks

系统中事件的时间用一个序号表示，这个序号代表量该事件发生的时间。对于每个P~i~定义了一个时钟函数C~i~，Ci是事件a发生的事件序号。由于Ci是一个逻辑时钟，和物理时钟没有关系。由于不依赖物理时钟，为了保证系统时钟的正确性，我们定义的时钟必须基于事件发生的先后顺序，即如果a先于b发生，那么a的时钟序号应该小于b的时钟序号，定义如下：

Clock Condition: 对于任意事件a，b，如果a⟶b，那么C < C

需要注意该命题的逆命题不成立。（图中C(p~3~) < C(q~3~)但是p~3~ ↛ q~3~）

可以看出关系⟶满足以下两个条件：

C1: 如果a和b是进程P~i~中的事件，并且a先于b发生，那么C~i~ < C~i~

C2: 如果a是进程P~i~发送消息的事件，b是进程P~j~接收消息的事件，那么C~i~ < C~j~

为了满足C1和C2系统需要遵守如下的规则：

IR1: 进程P~i~内的两个连续事件的序列号需要递增。

IR2: (a) a是进程P~i~发送消息的事件，b是进程P~j~接收消息的事件，T~m~=C~i~ (b) P~j~ 接收到消息时将C~j~设为大于T~m~以及高于自己当前时钟的值。

利用全序关系解决分布式资源互斥问题

考虑如下场景：一个资源一次只能被一个进程使用，进程之间必须使用同步措施以保证不会发生资源访问冲突。我们希望找到一种算法满足以下三个条件：

（i）已经获得资源的进程需要先释放，其他进程才能获得该资源。

（ii）资源授予进程的顺序需要按照他们的请求顺序。

（iii）如果每个获得资源的进程最终都能释放资源，那么每一个申请资源的进程最终都能获得资源。

使用一个中心的进程调度器按照资源请求的接收顺序授予资源将不能满足上述条件。例如，P~0~是中心调度进程，P~1~首先发送了一个消息给P~0~，然后发送了一个消息给P~2~，P~2~收到消息后又发送了一个消息给P~0~，那么P~2~发送的消息可能先于P~1~的消息到达P~0~，P~0~就会先将资源授予P~2~，这违反了条件(ii)。

为了解决上述问题，我们实现了一个满足全序关系的系统，系统中每个进程都维护一个request queue，初始时都包含T~0~:P~0~ request resource 。算法需要满足以下5个规则：

（1）P~i~请求资源时，首先给系统中所有其他进程发送一个T~m~:P~i~ request resource 的请求消息，并将该消息加入到自己的request queue中，T~m~是消息的时间戳。

（2）进程P~j~将收到消息T~m~:P~i~ request resource 加入自己的request queue，并给P~i~发送一个acknowledge message

（3）P~i~释放资源时，会将T~m~:P~i~ request resource 从自己的request queue中移除，并且给其他进程发送P~i~ release resource 的消息。

（4）进程P~j~收到 P~i~ release resource 的消息时，它会将自己request queue中的T~m~:P~i~ request resource 消息移除。

（5）当满足以下两个条件时，P~i~可以获得资源：（i） T~m~:P~i~ request resource 在队列中的全序关系最小 （ii）P~i~收到了来自其他进程的关于消息T~m~:P~i~的acknowledge message。

那么上述算法定义的5个规则是如何满足条件（i）（ii）（iii）的呢？

条件（i）：根据规则（3）、（4）我们可以知道，进程在释放资源时，会首先从自己的request queue中移除消息，然后通知其他进程移除消息，使得其他进程的消息变成全序最小的，最终可以获得资源。因而满足条件（i）。

条件（ii）：全序规则是由偏序规则扩展而来，因而可以满足偏序定义的顺序（原文限定，事件接收的顺序和发送的顺序一致），因而满足条件（ii）。

条件（iii）：规则（2）保证了规则（5）中的条件（ii）可以满足，规则（3）、（4）保证进程释放资源时，所有进程都会将该资源请求消息从队列中删除，其他进程的消息将会变成全序最小，使得条件（iii）可以满足。
